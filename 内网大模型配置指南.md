## 方案一：使用 OpenAI 兼容接口（推荐）

如果您的内网 DeepSeek/Qwen3 提供了 OpenAI 兼容的 API 接口，这是最简单的方案。

### 步骤 1：修改 `.env` 文件

在项目根目录的 `.env` 文件中添加/修改以下配置：

```bash
# 使用 OpenAI 客户端连接内网模型
OPENAI_API_KEY=your_internal_api_key
OPENAI_BASE_URL=http://your-internal-llm-server:port/v1

# 如果使用内网嵌入模型
DEEPWIKI_EMBEDDER_TYPE=openai

# 注释掉或删除 Google API Key（如果不再需要）
# GOOGLE_API_KEY=AIzaSyCZa9LCraXncT7foh0gsxTG3NUU5G3foKQ
# DEEPWIKI_EMBEDDER_TYPE=google
```

### 步骤 2：修改 `api/config/generator.json`

编辑 `D:\CodeLab\deepwiki-open\api\config\generator.json` 文件：

```json
{
  "default_provider": "openai",
  "providers": {
    "openai": {
      "default_model": "deepseek-chat",
      "supportsCustomModel": true,
      "models": {
        "deepseek-chat": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "deepseek-r1": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "qwen3-8b": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "qwen3-32b": {
          "temperature": 0.7,
          "top_p": 0.8
        }
      }
    }
  }
}
```

### 步骤 3：（可选）配置嵌入模型

如果内网也提供嵌入模型 API，编辑 `api/config/embedder.json`：

```json
{
  "embedder": {
    "client_class": "OpenAIClient",
    "model": "text-embedding-3-small",
    "dimensions": 1536,
    "encoding_format": "float",
    "api_key": "${OPENAI_API_KEY}",
    "base_url": "${OPENAI_BASE_URL}"
  }
}
```

## 方案二：使用 Dashscope 客户端（阿里云通义千问）

项目已经内置支持 Dashscope（阿里云通义千问），可以直接使用。

### 步骤 1：修改 `.env` 文件

```bash
# 通义千问 API Key
DASHSCOPE_API_KEY=your_dashscope_api_key

# 如果有自定义内网端点
DASHSCOPE_BASE_URL=http://your-internal-qwen-server:port

# 如果使用通义千问的嵌入模型
DEEPWIKI_EMBEDDER_TYPE=openai
OPENAI_API_KEY=your_dashscope_api_key
OPENAI_BASE_URL=http://your-internal-qwen-server:port/v1
```

### 步骤 2：修改 `api/config/generator.json`

```json
{
  "default_provider": "dashscope",
  "providers": {
    "dashscope": {
      "default_model": "qwen-plus",
      "supportsCustomModel": true,
      "models": {
        "qwen-plus": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "qwen-turbo": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "qwen3-8b": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "qwen3-32b": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "deepseek-r1": {
          "temperature": 0.7,
          "top_p": 0.8
        }
      }
    }
  }
}
```

### 步骤 3：确认 Dashscope 客户端配置

检查 `api/dashscope_client.py` 是否已存在（项目已包含），确认支持自定义 base_url。

## 方案三：使用本地 Ollama 部署

如果您在内网使用 Ollama 部署了 DeepSeek 或 Qwen3，适用此方案。

### 步骤 1：修改 `.env` 文件

```bash
# Ollama 服务器地址
OLLAMA_HOST=http://your-internal-ollama-server:11434

# 使用 Ollama 嵌入模型
DEEPWIKI_EMBEDDER_TYPE=ollama
```

### 步骤 2：修改 `api/config/generator.json`

```json
{
  "default_provider": "ollama",
  "providers": {
    "ollama": {
      "default_model": "deepseek-r1:8b",
      "supportsCustomModel": true,
      "models": {
        "deepseek-r1:8b": {
          "options": {
            "temperature": 0.7,
            "top_p": 0.8,
            "num_ctx": 32000
          }
        },
        "deepseek-r1:14b": {
          "options": {
            "temperature": 0.7,
            "top_p": 0.8,
            "num_ctx": 32000
          }
        },
        "qwen3:8b": {
          "options": {
            "temperature": 0.7,
            "top_p": 0.8,
            "num_ctx": 32000
          }
        },
        "qwen3:14b": {
          "options": {
            "temperature": 0.7,
            "top_p": 0.8,
            "num_ctx": 32000
          }
        }
      }
    }
  }
}
```

### 步骤 3：修改 `api/config/embedder.json`

如果使用 Ollama 的嵌入模型：

```json
{
  "embedder_ollama": {
    "client_class": "OllamaClient",
    "model": "qwen3:8b",
    "dimensions": 768,
    "encoding_format": "float",
    "host": "${OLLAMA_HOST}"
  },
  "retriever": {
    "top_k": 3,
    "mode": "default"
  },
  "text_splitter": {
    "split_by": "word",
    "chunk_size": 400,
    "chunk_overlap": 200
  }
}
```

### 步骤 4：确保 Ollama 服务器已部署模型

在 Ollama 服务器上执行：

```bash
# 拉取 DeepSeek 模型
ollama pull deepseek-r1:8b

# 拉取 Qwen3 模型
ollama pull qwen3:8b

# 拉取嵌入模型（如果需要）
ollama pull nomic-embed-text
```

## 完整配置步骤示例

假设您的内网有一个 DeepSeek API 服务器在 `http://192.168.1.100:8000`，以下是完整配置：

### 1. 修改 `.env`

打开 `D:\CodeLab\deepwiki-open\.env`，修改为：

```bash
# 内网 DeepSeek API 配置
OPENAI_API_KEY=sk-your-internal-key
OPENAI_BASE_URL=http://192.168.1.100:8000/v1

# 使用 OpenAI 兼容的嵌入模型
DEEPWIKI_EMBEDDER_TYPE=openai

# 注释掉 Google API（不再需要）
# GOOGLE_API_KEY=AIzaSyCZa9LCraXncT7foh0gsxTG3NUU5G3foKQ
# DEEPWIKI_EMBEDDER_TYPE=google
```

### 2. 修改 `api/config/generator.json`

打开 `D:\CodeLab\deepwiki-open\api\config\generator.json`，完全替换为：

```json
{
  "default_provider": "openai",
  "providers": {
    "openai": {
      "default_model": "deepseek-chat",
      "supportsCustomModel": true,
      "models": {
        "deepseek-chat": {
          "temperature": 0.7,
          "top_p": 0.8
        },
        "deepseek-r1": {
          "temperature": 0.7,
          "top_p": 0.8
        }
      }
    }
  }
}
```

### 3. （可选）修改 `api/config/embedder.json`

如果内网提供嵌入模型 API，打开 `D:\CodeLab\deepwiki-open\api\config\embedder.json`，修改嵌入模型配置：

```json
{
  "embedder": {
    "client_class": "OpenAIClient",
    "model": "text-embedding-3-small",
    "dimensions": 1536,
    "encoding_format": "float",
    "api_key": "${OPENAI_API_KEY}",
    "base_url": "${OPENAI_BASE_URL}"
  },
  "retriever": {
    "top_k": 3,
    "mode": "default"
  },
  "text_splitter": {
    "split_by": "word",
    "chunk_size": 400,
    "chunk_overlap": 200
  }
}
```

### 4. 重启后端服务

配置完成后，重启后端服务使配置生效：

**Windows 系统：**

```bash
# 1. 停止当前运行的后端服务（按 Ctrl+C）

# 2. 重新启动后端服务
cd D:\CodeLab\deepwiki-open
set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890
C:\Users\Administrator\AppData\Local\pypoetry\Cache\virtualenvs\open-deepwiki-api-CLxKQ3NS-py3.13\Scripts\python.exe -m api.main
```

**前端无需重启**（已在运行）

## 注意事项

### 1. API 兼容性

- **确保内网 API 格式兼容**：内网 DeepSeek/Qwen3 API 必须与 OpenAI API 格式兼容
- **检查端点路径**：确认 API 端点路径（通常是 `/v1/chat/completions`）
- **验证返回格式**：确认返回的 JSON 格式与 OpenAI 一致

### 2. 网络连接

- **确保后端能访问内网服务器**：测试网络连通性
- **检查防火墙规则**：确保端口未被防火墙阻止
- **VPN 或内网连接**：如需 VPN 连接内网，确保 VPN 已连接

### 3. 嵌入模型配置

- **建议使用同一提供商的嵌入模型**：获得最佳语义一致性
- **如无嵌入模型 API**：可以继续使用 Google AI 或 OpenAI 的公有云嵌入模型
- **嵌入向量维度**：确认嵌入模型的向量维度并正确配置

### 4. 代理设置

如果需要通过代理访问内网服务，在启动后端时设置：

```bash
set HTTP_PROXY=http://proxy-server:port
set HTTPS_PROXY=http://proxy-server:port
```

### 5. 性能优化

- **调整 `num_ctx` 参数**：根据模型上下文长度调整（Ollama）
- **调整 `temperature` 和 `top_p`**：根据生成质量需求调整
- **配置 `chunk_size`**：根据模型最大 token 数调整文本分块大小

### 6. 日志和调试

- **查看后端日志**：`api/logs/application.log`
- **启用 DEBUG 日志**：在 `.env` 中添加 `LOG_LEVEL=DEBUG`
- **检查 API 调用**：使用 Postman 或 curl 直接测试 API

### 7. 常见问题

**问题 1：连接超时**
```
解决方案：
1. 检查内网服务器地址和端口是否正确
2. 确认网络连通性（ping 测试）
3. 检查防火墙设置
```

**问题 2：API Key 无效**
```
解决方案：
1. 确认 API Key 格式正确
2. 检查内网服务是否需要特殊的认证方式
3. 查看服务器端日志
```

**问题 3：模型名称不匹配**
```
解决方案：
1. 确认内网服务支持的模型名称
2. 更新 generator.json 中的模型名称
3. 使用 supportsCustomModel: true 支持自定义模型名
```

**问题 4：生成质量不佳**
```
解决方案：
1. 调整 temperature 参数（降低随机性）
2. 增加 top_p 参数（提高多样性）
3. 检查模型是否正确加载
4. 考虑使用更大参数量的模型
```
